# 🎧 Music Streaming Churn Analysis

## 📌 Project Overview

This project focuses on analyzing **customer churn in a music streaming platform**.
The goal is to identify **why users stop using the service** and provide **data-driven insights** to improve retention.

---

## 🎯 Business Problem

Music streaming platforms face high user churn due to factors like buffering, pricing, and user experience.

👉 The objective of this project is to:

* Identify key factors influencing churn
* Analyze user behavior patterns
* Provide actionable recommendations to reduce churn

## 📂 Dataset Description

The dataset includes:

* User ID
* Subscription type
* Device type (Android, iOS, Web)
* Network type (WiFi, 4G, 5G)
* Buffering time
* Payment method
* Auto-renewal status
* Churn status

---

## 🔍 Key Analysis Performed

### 📈 1. Churn Rate Analysis

* Overall churn rate calculated
* Comparison across different segments

### 📱 2. Device-Based Churn

* Analyzed churn across Android, iOS, and Web users

### ⏱️ 3. Buffering Impact

* Higher buffering time → higher churn rate

### 🌐 4. Network Type Analysis

* Compared churn across WiFi, 4G, and 5G

### 🎵 5. Genre Preference Analysis

* Studied user engagement based on music genres

### 👥 6. Demographic Insights

* Gender and age group-based churn trends

---

## 📊 Key Insights

* ⚡ Users with **high buffering time** show higher churn
* 📉 **Auto-renewal OFF users** are more likely to churn
* 📱 Device type has minimal variation but still impacts engagement
* 🌐 Network quality directly affects user retention
* 🎧 Certain genres have higher engagement and lower churn

---

## 💡 Recommendations

* Improve streaming performance (reduce buffering)
* Encourage auto-renewal with incentives
* Optimize app performance across all devices
* Provide personalized music recommendations
* Improve network adaptability

This analysis helps businesses:
